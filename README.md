# 02-미니 인재풀 시스템

## 개요

A 유저가 자신의 정보를 등록하면, B 유저가 A 유저 상세 페이지에서 **‘PICK’** 버튼을 눌러 **A 유저와 1:1 채팅을 시작**할 수 있는 시스템

#### **핵심 흐름(User Flow)**

1. A 유저가 자기 정보를 등록한다.
2. B 유저가 리스트에서 A 유저의 요약 정보를 확인한다.
3. B 유저가 A 유저 상세 페이지로 진입한다.
4. 상세 정보를 확인한 뒤 하단 **PICK** 버튼을 누른다.
5. A 유저와 대화 가능한 **1:1 채팅방**으로 연결된다.

## 목적

#### **만들려는 이유(배경)**

회사 앱에 도입하려는 기능을 말로만 설명하면 의도(UX/느낌)가 정확히 전달되지 않는다.
실제로 사용 가능한 미니 버전(프로토타입)이 있으면, 내가 생각한 사용감과 흐름을 더 명확하게 전달할 수 있을 것으로 기대한다.

#### **성공 기준(Success Criteria)**

* 유저가 자기 정보를 **정상적으로 등록/수정**할 수 있다.
* **1:1 채팅 화면까지 정상적으로 동작**한다.
* 웹이지만 **모바일 UI/UX 형태**로 제공된다.

## 기능 요구사항

### 필수(Must)

1. **간단 로그인(유저 구분: A/B 포함)**
   * 입력: **이름 + 비밀번호**
   * 규칙
     * 이름이 존재하고 비밀번호가 일치하면 로그인 성공
     * 이름이 없으면 해당 이름으로 **계정 자동 생성 후 로그인**
     * 이름은 있는데 비밀번호가 불일치하면 **“비밀번호가 틀렸습니다.”** 문구 표시
2. **유저 프로필 관리**
   * 유저 정보 **등록/수정/삭제/조회(CRUD)**
3. **1:1 채팅**
   * B 유저가 A 유저 상세에서 **PICK** 시 채팅방 연결
